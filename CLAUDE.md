# CLAUDE.md — конвенции проекта «Джарвис»

> Этот файл Claude Code читает автоматически в каждой сессии. Здесь — durable-правила проекта, не разовая задача.

## Что это

Локальный, оффлайн, асинхронный голосовой ассистент на Kali Linux. Архитектура — Event-Driven Microservices поверх Mosquitto MQTT (`localhost:1883`). Каждый компонент — независимый демон, общается только JSON-сообщениями в MQTT-топиках. Ни один процесс не блокирует другие.

## Железо (бюджет ресурсов)

Intel N100, без дискретного GPU, **8 ГБ RAM**, KDE Plasma на **Wayland**. Инференс LLM — на CPU. RAM в постоянном дефиците: не держать в памяти лишнего, не загружать ненужные модели, у Ollama настроен `keep_alive`. Любое решение проверять на «влезет ли в 8 ГБ вместе со всем остальным».

## Главный принцип: модульность как LEGO

Добавление нового модуля и его подключение к шине = **один новый файл + несколько строк**, без правки существующих модулей.

- Вся MQTT-обвязка, переподключение, логирование и graceful shutdown живут в базовом классе **`JarvisModule`** (`jarvis_bus.py`).
- Каждый модуль — наследник `JarvisModule`, содержащий ТОЛЬКО свою логику (на какие топики подписан, что делает).
- Нельзя дублировать MQTT-/logging-обвязку в модулях. Если её не хватает — расширяй базовый класс, а не копипастишь.

## Контракты шины (MQTT-топики)

| Топик            | Payload                                                   | QoS |
|------------------|-----------------------------------------------------------|-----|
| `jarvis/input`   | `{"text": "..."}`                                         | 0   |
| `jarvis/execute` | `{"command_tag": "..."}`                                  | 1   |
| `jarvis/say`     | `{"text": "...", "source": "module"}`                     | 0   |
| `jarvis/state`   | `{"state": "idle|listening|thinking|speaking", "source"}` | 0   |

Менять контракты — только осознанно и с обновлением этой таблицы и всех модулей.

## Команды OS-агента

- Карта команд — в `commands.yaml`, НЕ в коде. Формат: `тег → {описание, команда (список аргументов), ждать_вывод}`.
- Добавить/изменить команду = правка `commands.yaml`, без изменения Python-кода.
- Запуск только `subprocess.Popen`, `shell=False`, список аргументов. **Никогда** не подставлять пользовательский текст в команду — только lookup тега. Неизвестный тег → лог + предупреждение в `jarvis/say`, не выполнять.

## Стек и версии

- **paho-mqtt 2.x** — `mqtt.Client(mqtt.CallbackAPIVersion.VERSION2)`, новые сигнатуры колбэков.
- **pydantic v2**.
- STT: `sherpa-onnx` (silero-VAD + SenseVoice-Small), захват `sounddevice` 16 кГц моно. Wake-word «джарвис» ловится через VAD+ASR (готового русского KWS нет); детекцию держать заменяемой на openWakeWord.
- LLM: Ollama, `qwen2.5:0.5b-instruct` (instruct, не coder), structured output через `format=JarvisResponse.model_json_schema()`.
- TTS: **Piper** (ONNX), голос `ru_RU dmitri`, путь в `config.py` (позже — дообученный голос того же формата).
- **API библиотек сверять с установленными версиями** (`pip show`, docstrings, `--help`), не писать по памяти.

## Надёжность

- Автопереподключение MQTT (`on_disconnect` + exponential backoff) — в `JarvisModule`.
- Все внешние вызовы (Ollama, subprocess, аудио, модели) — в `try-except` + `logging`. Демон не падает никогда.
- Логи — `RotatingFileHandler` в `logs/`, имя файла = имя модуля.
- Graceful shutdown по `SIGTERM`/`SIGINT` — в `JarvisModule`.
- Автозапуск — systemd **`--user`**-юниты (аудио PipeWire и Wayland-сессия живут в пользовательском сеансе), `Restart=always`.

## Стиль и процесс

- Комментарии и docstrings — **на русском**.
- Ветка — `main`.
- Сложные задачи начинать с плана; модули добавлять по одному, проверяя через `mosquitto_pub`/`mosquitto_sub`.
- HUD-оверлей — это Wayland/KWin: click-through и always-on-top ограничены протоколом, поведение проверять под установленными Qt/PySide6/KWin, иметь запасной вариант. Без тяжёлых GPU-эффектов.

## Как добавить новый модуль (шпаргалка)

1. Создать `jarvis-<имя>.py`, унаследовать `JarvisModule`.
2. В `on_start()` подписаться на нужные топики через `self.subscribe(...)`.
3. Запустить через `JarvisModule.run()`; при необходимости добавить `--user`-юнит в `systemd/`.
