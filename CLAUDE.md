# CLAUDE.md — Джарвис (локальный голосовой пульт)

## 📌 ЧТО ЭТО И АППАРАТНЫЕ ОГРАНИЧЕНИЯ
«Джарвис» — простой, надёжный, **полностью локальный** голосовой пульт на базовые команды (event-driven микросервисы через MQTT). 
* **Железо:** Intel N100 (4 ядра), без GPU, 8 ГБ RAM, swap = 0. Слабое железо!
* **ОС:** TUXEDO OS 24.04 (Ubuntu-база), KDE Plasma 6, Wayland.
* **Главное правило:** НИКАКИХ LLM и облаков. Система должна потреблять минимум ресурсов.
* **Язык проекта:** Строго русский (код, логи, коммиты, ответы). Ответы должны быть «в характере» (сухая ирония, вежливый ИИ-дворецкий).

## 🏗 АРХИТЕКТУРА И СТЕК (Локальный офлайн)

1. **STT «Уши»:** `sherpa-onnx` + `zipformer-ru-small` + `silero-VAD`. Захват через `sounddevice` (16кГц). 
   * *Важно:* 1 поток (`num_threads=1`) работает быстрее на N100. Push-to-talk (PTT) через правый Ctrl (чтение `/dev/input`).
2. **CORE «Мозг»:** `jarvis/services/core.py`. 
   * Распознавание (`matcher.py`): **Слой 1 правил** (синонимы: точное совпадение O(1) + difflib для искажений STT) → **Слой 2 семантики** (`rubert-tiny2` ONNX INT8, грузится лениво; mean-pool 312-вектор → **линейный классификатор LogReg**). 
   * *Классификатор:* обучается на синонимах+примерах из `commands.yaml`, веса кешируются (`matcher_clf.npz`); инференс — `softmax(W·x+b)` в **чистом numpy** (sklearn в рантайме НЕ грузится, нужен лишь при переобучении после правки YAML). Пороги `clf_threshold`/`clf_margin` в `settings.yaml`. Сбой/нет sklearn → fallback на косинус-kNN; нет эмбеддера → только правила. Бенч/тюнинг: `tools/bench_matcher.py`.
   * *INT8-эмбеддер:* `model_int8.onnx` (~30 МБ против 116 МБ fp32) генерируется `jarvis models --quantize` (`tools/quantize_embedder.py`); `config` предпочитает int8 при наличии. Кеши инвалидируются по `stat` файла модели автоматически.
3. **TTS «Голос»:** `Silero` (спикер **eugene**, `v4_ru.pt`) с **кэшем целых фраз**.
   * *Архитектура:* Silero — PyTorch (`.pt`, ONNX нет), синтез на N100 ~реального времени (RTF≈1) — слишком медленно для горячего пути. Поэтому ВСЕ фразы заранее рендерятся в **WAV-кэш** (`jarvis tts build`, кэш в `cache/tts/<voice_id>/<dsp_sig>/`, ключ = SHA1 итогового текста) с офлайн «JARVIS-DSP» (ffmpeg: понижение тона, тёплый EQ, компрессия, лёгкий реверб — «дворецкий»). Рантайм играет готовый WAV **без torch и без задержки** (`jarvis/tts_cache.py`, `jarvis/tts_engine.py`, `jarvis/tts_dsp.py`).
   * *Свободный текст* (надиктовка, незнакомые города) — ленивый синтез на промахе: torch грузится в фоне, синтезирует, кладёт в кэш, выгружается по простою (`silero_unload_after`). torch — опциональный extra `[silero]`, в горячий путь НЕ импортируется. Филлер «Сек+унду, сэр.» из кэша заполняет паузу холодного синтеза.
   * *Ударения:* Silero чтит `+` перед ударной гласной (в отличие от Piper). Фразы размечены `+`; ручной словарь омографов — `voice.stress`; авто-ударения свободного текста (`ruaccent`) — опц. `voice.auto_stress`. Валидатор — `jarvis doctor` (`check_stress_marks`).
   * *Подбор тембра:* `python tools/voice_studio.py` — веб-UI с живыми ползунками DSP (synth-once/DSP-many, мгновенное A/B), кнопка сохранения пресета в `settings.yaml`.
   * *Piper* остаётся аварийным фоллбэком (extra `[piper]`, onnxruntime без torch): синтез, если Silero/torch недоступны.
   * *Критично:* Воспроизведение ТОЛЬКО через `pw-cat` (PipeWire) в subprocess. `sounddevice` ломается на TUXEDO HDMI-выходах (немота). Готовый PCM стримится чанками — огибающая ламп, ducking и barge-in работают как при потоковом синтезе.
4. **OS-агент «Руки»:** Исполняет теги из `commands.yaml`. 
   * *Критично:* `shell=False`, строго через списки аргументов (allow-list). Произвольный shell запрещён.
5. **Шина:** `paho-mqtt` (топики: `jarvis/input`, `jarvis/execute`, `jarvis/say`, `jarvis/state`, `jarvis/lamp`, `jarvis/event`). Наследники `JarvisModule`.

## 📁 СТРУКТУРА ПРОЕКТА
* `settings.yaml` — ЕДИНЫЙ файл всех настроек. Код читает его через `config.py`.
* `commands.yaml` — Карта команд (тег → команда ОС, синонимы, ответы). 
* `schedule.yaml` — Расписание будильников/таймеров (пишется кодом атомарно).
* `jarvis/bus.py` — Базовый класс `JarvisModule` (MQTT, логи, shutdown).
* `jarvis/config.py` — Загрузка настроек. **Все пути резолвятся от `BASE_DIR` в абсолютные**.
* `jarvis/phrases.py` — Механизм выбора фраз для ответов без повторов.
* `jarvis/services/` — Микросервисы (core, stt, tts, os_agent, activity_monitor, scheduler, lamp, phone).

## ⚠️ ЖЁСТКИЕ ПРАВИЛА И ИЗВЕСТНЫЕ ГРАБЛИ (НЕ ПОВТОРЯТЬ ОШИБОК)

1. **Пути файлов:** В `settings.yaml` пишутся относительные пути, но под systemd они теряются. В коде ВСЕ пути прогонять через `config._get_path`, чтобы они стали абсолютными от корня проекта.
2. **Никаких `shell=True`:** Безопасность превыше всего. Все команды в `commands.yaml` пишутся списками (allow-list).
3. **Направление (вкл/выкл) — только правилами:** Эмбеддинги (`rubert-tiny2`) не кодируют полярность, **и классификатор Слоя 2 это наследует** (учится на тех же векторах). Антонимы вкл/выкл, громче/тише ОБЯЗАНЫ матчиться слоем правил (точные синонимы + `difflib`); Слой 2 их лишь страхует через `clf_margin` (близкие вероятности → переспрос). Регрессию ловит `jarvis doctor` (`check_antonyms`). Никогда не ждите, что классификатор «научится» различать антонимы.
4. **Умные лампы (Tuya):** Версия протокола 3.5. Управление по DPS, `nowait=False`. TCP простаивает и рвётся, поэтому реализован тихий ping каждые 20 сек.
5. **Wayland и Idle:** `GetSessionIdleTime` не работает. Активность пользователя мониторится прямым чтением `/dev/input/event*` (нужна группа `input`).
6. **Задержки TTS:** Голос — из WAV-кэша (мгновенно), синтез Silero в горячий путь НЕ попадает. torch НИКОГДА не импортировать на уровне модуля `tts.py` — только лениво внутри `SileroEngine._load()` (иначе systemd-старт TTS потянет ~400 МБ RAM при swap=0). После правки фраз/DSP/голоса — пересобрать кэш (`jarvis tts build`); смена `voice.dsp`/спикера меняет сигнатуру → старый кэш осиротеет (`jarvis tts stats --prune`).
7. **Перебивание (Barge-in):** STT шлёт `jarvis/tts/control {"action":"stop"}`. TTS обрывает `pw-cat` (terminate), это штатное поведение, а не сбой.
8. **Ударения и кэш-ключ:** Ключ кэша = SHA1 ИТОГОВОГО текста (после `apply_pronunciation` + `apply_stress`). Хардкод-строки сервисов и `HARDCODED` в `tts_prerender.py` должны нести ТЕ ЖЕ `+`-ударения, что реально шлются, иначе ключ не сойдётся и фраза уйдёт в ленивый синтез. `+` ставится строго ПЕРЕД ударной гласной (валидатор `check_stress_marks`).

## 🚀 КОМАНДЫ (CLI)
* `jarvis start` — поднять/перезапустить.
* `jarvis live` — TUI панель состояния.
* `jarvis status` / `jarvis doctor` — проверка здоровья системы и конфигов (вкл. эмбеддер, антонимы, классификатор Слоя 2, голос Silero, покрытие кэша, ударения, ffmpeg).
* `jarvis models --download silero_eugene` — голос Silero; `jarvis models --quantize` — INT8-эмбеддер.
* `jarvis tts build [--only static|dynamic] [--force]` — пред-рендер фраз в WAV-кэш; `jarvis tts stats [--prune]` — покрытие/объём/чистка.
* `jarvis say "..."` — произнести фразу голосом Джарвиса (проверка голоса/DSP/ударений; хит кэша мгновенно).
* `python tools/voice_studio.py` — веб-студия подбора тембра (живые ползунки DSP, A/B, сохранение пресета).
* `pip install -e .` — entry points. Голос Silero: `pip install -e '.[silero]' --extra-index-url https://download.pytorch.org/whl/cpu` (torch CPU). Системно нужен `ffmpeg`.
