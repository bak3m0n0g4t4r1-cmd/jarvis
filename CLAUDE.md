## ЧТО ЭТО

Голосовой ассистент «Джарвис»: event-driven микросервисы через MQTT.
Цель — не «голосовой пульт», а умный ассистент в духе J.A.R.V.I.S.: понимает с
полуслова, рассуждает, управляет системой, отвечает в характере.

**Язык проекта — русский.** Код, комментарии, логи, ответы ассистента, сообщения
коммитов — на русском. Общение с пользователем — на русском.

## ЖЕЛЕЗО И ОКРУЖЕНИЕ (важно для всех решений)

- **Kali Linux**, KDE Plasma на **Wayland**, оболочка **zsh**.
- Пользователь занимается пентестом — НЕЛЬЗЯ гнать системный трафик через прокси/VPN.

- venv в `.venv` (`source .venv/bin/activate`). PEP 668 — pip ТОЛЬКО в venv, без `--break-system-packages`.
- Пользователь в России → геоблоки Google/Anthropic (см. «Прокси»).

## АРХИТЕКТУРА «ТЕЛО / МОЗГ» (ключевое решение)

N100 — «ТЕЛО», облако — «РАЗУМ». Принято осознанно: умный Джарвис ТРЕБУЕТ интернета;
офлайн = деградированный режим (базовые команды + заглушки в характере).

```
Пользователь говорит
   │
   ▼
ЛОКАЛЬНАЯ 1.5B (qwen2.5:1.5b-instruct) — ПРИВРАТНИК/ДИСПЕТЧЕР
   ├─ узнала базовую команду (жёсткий паттерн) → выполнить ЛОКАЛЬНО (быстро, офлайн, Wi-Fi-safe)
   └─ не базовая / сложное / беседа / вопрос → в ОБЛАКО
                                                  │
                                                  ▼
                              GEMINI 3.5 FLASH — «МОЗГ» (через US-прокси)
                              — function calling по allow-list
                              — grounding (свежие факты: погода/новости/курсы)
                              — характер Джарвиса (system instruction)
                              — ответ/действие → тело (N100)
```

**Почему 1.5B только диспетчер:** на свободной беседе 1.5B (и 3B) тупит и тормозит —
проверено. Сужена до классификации интентов + коротких подтверждений команд. Ум — в облаке.

**Почему базовые команды локальные:** (1) скорость и офлайн; (2) КРИТИЧНО — команды
вроде «выключи Wi-Fi» НЕЛЬЗЯ гнать через облако, иначе Джарвис, выполняя их, убьёт себе
связь с мозгом. Потолок ~100, жёсткие паттерны, без «чудес» от модели.

## СТЕК (работает на железе, подтверждено `jarvis doctor`)

- **STT «Уши»**: sherpa-onnx, модель **zipformer-ru** (`sherpa-onnx-small-zipformer-ru-2024-09-18`,
  offline transducer, int8). НЕ SenseVoice (без русского). silero-VAD. Захват
  sounddevice 16кГц моно. Wake-word «джарвис» через VAD+ASR + difflib (ловит «джарвиз/сервис»).
  VAD-пороги ЗАКРЕПЛЕНЫ: `JARVIS_VAD_THRESHOLD=0.3`, `JARVIS_VAD_MIN_SILENCE=0.8` — НЕ ТРОГАТЬ.
- **Core «Мозг»**: Ollama, `qwen2.5:1.5b-instruct` (НЕ coder, НЕ 0.5b — тупят).
  Диспетчер интентов, structured output через `format=schema`.
- **Casual-бэкенд**: `jarvis/casual.py` — google-genai (2.7.0), `gemini-3.5-flash`,
  grounding, память ~8 реплик, характер, офлайн-фоллбэк, переключение gemini|local.
- **TTS «Голос»**: Piper 1.4.2, `ru_RU-dmitri-medium.onnx` (22050 Гц). API: `synthesize()`
  → итератор AudioChunk (`audio_int16_*`, `sample_rate`). НЕ старый `synthesize_stream_raw`.
- **OS-агент «Руки»**: исполняет теги из `commands.yaml`. `shell=False`, allow-list,
  без произвольного исполнения.

**MQTT-топики:** `jarvis/input` {"text"} · `jarvis/execute` {"command_tag"} QoS1 ·
`jarvis/say` {"text","source"} · `jarvis/state` {"state": idle|listening|thinking|speaking}.

**Лего-модульность:** новые модули наследуют `JarvisModule` из `bus.py`. Не дублировать
MQTT/логирование/shutdown — всё в базовом классе.

## ПРОКСИ ДЛЯ GEMINI (геоблок — обязательно знать)

Gemini API НЕ работает из РФ напрямую: `400 FAILED_PRECONDITION: User location is not
supported`. DNS-разлочка НЕ помогает (Google видит исходящий IP). Решение — HTTP/SOCKS5
**прокси с зарубежным IP**, проброшенный ТОЛЬКО в google-genai клиент.

- Прокси через `.env`: `JARVIS_GEMINI_PROXY` (пусто = напрямую).
- Затрагивает ТОЛЬКО Gemini-клиент в `casual.py`. Ollama (localhost), MQTT, STT — НАПРЯМУЮ.
  НЕЛЬЗЯ гнать системный/пентест-трафик через прокси.
- Таймаут Gemini: МИНИМУМ API 10с; стоит `JARVIS_GEMINI_TIMEOUT=15`. `HttpOptions.timeout` — в МИЛЛИСЕКУНДАХ (сверено на 2.7.0).

## БЕЗОПАСНОСТЬ (жёсткие правила)

- **НЕТ произвольному shell** для облачной модели (пока). Только предопределённые
  безопасные функции из allow-list. Модель ошибается/галлюцинирует — на Kali с правами
  это катастрофа.
- **Необратимые действия** (удаление, выключение, разрыв сети) — ТОЛЬКО после явного
  подтверждения голосом/кликом.
- **Зрение — только по явному запросу** («посмотри/помоги»). Никаких фоновых скриншотов.
- `commands.yaml` — только безопасные обратимые действия. Без root-деструктива/атак.

## ДОРОЖНАЯ КАРТА (ЭТАПАМИ, не монолитом)

Состояние: голосовое ядро (Фаза 1) работает. Заход 2 (Gemini casual) на диске.
Прокси для Gemini — в процессе.

1. **Этап 1 — Мозг с function calling** (важнейший). Перестройка core: Gemini как
   мыслитель, вызывающий локальные действия (управление + чтение состояния системы).
   Превращает пульт в агента.
2. **Этап 2 — Ротация ключей.** 5 ключей с РАЗНЫХ аккаунтов, авто-failover при `429 RESOURCE_EXHAUSTED`.
3. **Этап 3 — Расширение инструментов + подтверждения** необратимых действий.
4. **Этап 4 — Зрение.** Скриншот по запросу → Gemini мультимодально.
5. **Этап 5 — HUD.** Киношный визуал поверх готового интеллекта (последним). Веб-стек
   в прозрачном окне, чисто визуальный (без click-through — обход боли Wayland).
   Концепты: дыхание по краю + орб + диалоговая карточка + печатная машинка +
   звуковая волна + тепловая реакция на голос + док-режим.

Каждый этап — отдельный заход, проверка на железе перед следующим.

## РАБОЧИЕ ПРИНЦИПЫ

- **Plan mode для всего нетривиального.** План → подтверждение → код.
- **Один заход = одно связное изменение.** Не монолит. Узкая чёткая задача лучше расплывчатого куска.
- **СВЕРЯЙ API НА МАШИНЕ, не по памяти.** Версии меняются. Спасало многократно
  (таймаут мс/сек; метод Piper; способ прокси в SDK). Трогаешь внешнюю либу/модель —
  сперва проверь сигнатуру на установленной версии.
- **Не верить «готово» без проверки.** `jarvis doctor` зелёный (FAIL=0; WARN ок).
  Отчёт расходился с реальностью — проверять вживую.
- **Сервисы не перезапускать самому** — это проверка пользователя.
- **Обязательно `git commit` + `git push`** в конце захода. Браузерный CC раньше пушил
  в свои ветки и работа «терялась» — проверять `git branch -a`, пушить явно.
- Команды под KDE/Wayland проверять реально (бинарь в `$PATH` ≠ рабочий эффект).
